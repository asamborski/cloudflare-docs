[Access for Infrastructure](/cloudflare-one/access-controls/applications/non-http/infrastructure-apps/) now supports multi-factor authentication (MFA) for SSH connections. Administrators can require users to authenticate with a YubiKey PIV key before connecting to an SSH server, protecting against stolen WARP credentials and non-interactive SSH attempts.

When MFA is required, the SSH proxy prompts the user for public key authentication using their enrolled PIV key. The user touches their YubiKey (and enters their PIN if required by the key's policy) to prove possession. The proxy then generates a short-lived certificate and completes the connection to the upstream server. No changes are required to the SSH server configuration.

## Configure MFA for SSH

You can enforce MFA at the infrastructure application level or at the policy level. Policy-level settings override application-level settings, allowing you to require stricter MFA for sensitive usernames (for example, `root`) while applying different settings for other usernames.

You can also configure an MFA session duration. After the session expires, users must re-authenticate with their PIV key on the next SSH connection. Existing SSH sessions are not affected by session expiration.

For setup instructions, refer to [Enforce MFA](/cloudflare-one/access-controls/policies/mfa-requirements/#infrastructure-applications).

## Enroll a PIV key

Users enroll their YubiKey PIV key as an SSH MFA authenticator through the [App Launcher](/cloudflare-one/access-controls/access-settings/app-launcher/). During enrollment, the user uploads their PIV attestation certificates (`leaf.pem` and `intermediate.pem`), and Access extracts the SSH public key for future authentication. Users must also configure their SSH client to use a PIV-aware agent such as `yubikey-agent`.

For more information, refer to [Enroll a PIV key for SSH](/cloudflare-one/access-controls/access-settings/independent-mfa/#enroll-a-piv-key-for-ssh).
